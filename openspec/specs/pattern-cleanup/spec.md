# pattern-cleanup 规范

## 目的
定义 `bead-core` 的 `BeadPattern` 级**空间后处理**能力族——在配色/减色**之后**、统计/渲染**之前**，对最终 `BeadPattern` 做纯整数、确定、**永不发明色**的空间清理。首个成员：**连通域去斑**（`despeckle`——把小同色 4-连通分量按输入快照并入多数邻色），opt-in、默认关；未来的最小岛屿/平滑等空间清理归入此族。均为**库/复用原语、非生成入口**（由 `pipeline::generate_pattern` 选择性调用，见 pipeline 规范）。
## 需求
### 需求:连通域去斑（小同色分量并入多数邻色，输入快照语义）
`bead-core` MUST 提供 `BeadPattern` 级的**连通域去斑** `despeckle(pattern: &BeadPattern, min_region: u32) -> BeadPattern`（同形状、pattern→pattern、reuse 原语、**非**生成入口）。给定 `min_region`，把 `cells` 中**同色 4-连通分量**里珠数 ≤ `min_region` 的分量，整块**重映射到该分量的多数邻色**。

**执行模型 MUST 为输入快照（order-independent）**：分量发现、每个分量的珠数、以及多数邻色计票**全部对输入 `BeadPattern` 的快照**进行；所有重映射写入**独立的输出缓冲**；**单遍**——先前分量的重映射**不影响**后续分量的邻色计票（故结果与扫描顺序**无关**、序不变，`despeckle` 的输出由输入拓扑唯一决定）。这与「并入色是**输入中已存在**的相邻珠色」措辞自洽。

**多数邻色 MUST 逐边界邻接边计票**：对该分量的每个格，看其 4-邻中**异色**（不在本分量内）的外侧格——每条这样的**跨界邻接边计一票**给该外侧格的珠色（同一邻色被多条边接触则累加多票）；取**票数最多**的珠色为目标；平局取**调色板最小下标**。同色邻居按定义已在分量内、不计票。并入色 MUST 是**输入中已存在的相邻珠色**（**永不发明**珠板外色），故输出 `cells` 每个值仍是合法调色板下标。

`despeckle` 是**全函数**——前置条件：`cells` 均为合法调色板下标（同 `match_pattern` 输出后置条件）；合法输入下**不 panic**。**空 pattern**（`cells.len()==0`）原样返回、不 panic。分量**无异色邻居**（仅当**全图单色**）→ 该分量原样保留（无目标可并）。

#### 场景:小孤立点被并入多数邻色
- **给定** 一个 `BeadPattern`，某格是被同一背景色 4-邻包围的**单颗异色**（连通分量珠数 = 1）
- **当** 以 `min_region >= 1` 去斑
- **那么** 该格被重映射为背景色（多数邻色），输出同形状、`cells` 均合法下标、无新珠板外色

#### 场景:两相邻异色单点分量按输入快照并入、序不变
- **给定** 行片段 `[B, X, Y, B]`（上下均为背景 `B`），`X`、`Y` 为相邻的两个不同异色单点、各自珠数 = 1，`min_region >= 1`
- **当** 去斑
- **那么** `X` 与 `Y` 的邻色计票**均对输入快照**：`X` 的多数邻色为 `B`（`Y` 仅 1 票、`B` 多票）→ `X→B`；`Y` 同理 → `Y→B`；结果 `[B, B, B, B]`，且**与行优先/列优先扫描顺序无关**（先处理 `X` 不改变 `Y` 看到的邻色）

#### 场景:大于阈值的分量保留
- **当** 某同色 4-连通分量珠数 > `min_region`
- **那么** 该分量**不**被清理，逐格保留

#### 场景:目标平局取最小下标
- **给定** 一个 ≤`min_region` 的小分量，其边界两侧邻色**按边计票**后票数相等
- **当** 去斑
- **那么** 并入**调色板下标较小**者，重复调用一致

#### 场景:全图单色 / 空 pattern 原样返回
- **当** 对全图单色（分量无异色邻居）或 `cells.len()==0` 的 `BeadPattern` 去斑
- **那么** 原样返回、**不 panic**（无目标可并、无内容可清）

### 需求:去斑确定性（纯整数、跨架构位精确）
去斑 MUST 确定：输入快照语义（分量发现 / 计票读输入、写独立输出）、固定行优先扫描找连通分量、固定 4-连通、逐边界邻接边计票、平局取最小下标；**禁止**随机、`rayon`、`HashMap`/`HashSet` 迭代顺序泄漏、浮点。因全程整数（下标与计数），同一 `(BeadPattern, min_region)` 跨架构（arm64 / x86_64）**必须逐字节一致**——据此可钉一份**跨架构位精确**的整数 golden（区别于经 Oklab/Triangle 的 f32 同机档）。

#### 场景:重复去斑逐字节相等且跨架构位精确
- **当** 对同一 `(BeadPattern, min_region)` 多次去斑
- **那么** 每次输出完全相等；且因全程整数、输入快照序不变，输出等于硬编码期望 `cells`，该断言在 arm64 与 x86_64 上都通过
